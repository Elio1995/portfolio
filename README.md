# Portfolio — Elio Mehmeti

> Single-page Next.js portfolio. Editorial / engineer's-notebook aesthetic: serif display headings (Fraunces), warm cream + ink palette, saturated amber accent, mono section labels in the left rail. Deliberately escapes the 2024–25 "Inter + cool blue" templated default.

Live site: <https://elio-mehmeti.dev>

---

## What's in it

| Section | What it shows |
| --- | --- |
| Hero | Name, role, status pill, signature stack, primary CTAs (email, résumé, GitHub, LinkedIn) |
| Projects | 8 projects ordered by hiring impact — AI-native + mobile first, work entries last |
| Experience | Reverse-chronological job timeline with concise bullets |
| About | Three-paragraph bio + skills matrix |
| Contact | Email + social links + résumé download |

Plus the bits that make it shareable:

- **Dynamic Open Graph image** rendered at the edge (`app/opengraph-image.tsx`) so link previews on LinkedIn / Twitter / Slack look intentional, not default
- **Dynamic favicon** matching the palette (`app/icon.tsx`)
- **`robots.txt`** and **`sitemap.xml`** generated by Next.js metadata routes
- **Light + dark themes** with warm palettes both ways

## Editing content

All copy and links live in **`lib/data.ts`** — name, tagline, projects, experience, skills. Edit there; components re-render automatically.

**To add a project:** append to the `projects` array.

```ts
{
  name: "Project name",
  kind: "Personal", // or "Work"
  description: "One short paragraph. State what it does and the most interesting design choice.",
  stack: ["TypeScript", "Next.js 15", "Prisma"],
  image: "/screenshots/project-name.png", // optional — falls back to a typographic placeholder
  githubUrl: "https://github.com/Elio1995/project-name",
  liveUrl: "https://...",                  // optional
  liveLabel: "Custom label",               // optional override for the Live link
}
```

Drop a screenshot at `public/screenshots/<name>.png`. 16:9 aspect ratio renders best.

**To swap the résumé:** replace `public/resume.pdf` with a new file of the same name.

## Customizing the theme

Open `app/globals.css`. Two blocks matter:

1. **`@theme` block** — font family variables (Fraunces / Inter / JetBrains Mono).
2. **`:root` and `.dark` blocks** — HSL color tokens (`--background`, `--foreground`, `--accent`, etc.).

The accent is currently a saturated amber (`22 84% 40%` in light, `28 90% 60%` in dark). Swap to any other HSL triple to re-skin without touching the components.

The `.font-display` class drives the editorial serif look — using it on `<em>` makes the italic real, not slanted.

## Deploy to Vercel

Zero config. Two options:

### Option A — CLI (one shot)

```bash
pnpm dlx vercel               # First deploy — links the project
pnpm dlx vercel --prod        # Promote to production
```

### Option B — GitHub import

1. Push this repo to GitHub.
2. Open <https://vercel.com/new>.
3. Import the repo. Vercel auto-detects Next.js — accept defaults.
4. Click **Deploy**.

No environment variables required. First build takes ~45 seconds; subsequent builds are typically under 30.

### Custom domain

In the Vercel dashboard → Project → **Settings** → **Domains** → add `elio-mehmeti.dev` (or your own). Update the `SITE_URL` constant in `app/layout.tsx`, `app/opengraph-image.tsx`, `app/robots.ts`, and `app/sitemap.ts` to match.

## Visual design decisions

A short read on why the site looks the way it does, in case you want to riff:

- **Serif display + sans body + mono labels.** Three typefaces, each doing one specific job — display for emphasis, sans for prose, mono for metadata. Almost every templated portfolio uses one typeface for everything, which is why they look identical.
- **Warm cream over off-white.** `hsl(40 25% 98%)` reads as paper-like, not screen-like. Distinguishes from the standard `#ffffff` default.
- **Saturated amber, not cool blue.** Blue is the 2020–2024 default accent for every Tailwind starter ever shipped. Amber is uncommon, warmer, and feels hand-picked.
- **Mono section labels in the left rail.** Asymmetric layout pattern borrowed from print magazines. Adds rhythm to vertical sections and signals deliberate composition.
- **№ 01, № 02, …** Roman numeral abbreviation. Niche but immediately legible. Reads as "this person noticed Unicode."
- **`<em>` inside `.font-display` is a real italic.** Fraunces ships with a true italic master, not a slanted version. The hero name uses `<em>` on the surname to demonstrate.
- **No "Built with Next.js · Vercel" footer.** Anyone who reaches that line can see what it's built with. Crediting the framework is a tell.
